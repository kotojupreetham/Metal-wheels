## 3. Video Showcase Production Plan

Videos tell a story that static images cannot, especially since your site uses **autoplay looping MP4 videos** for each car card, which is an impressive design choice!

We suggest creating and hosting three promotional videos:

### A) The 30-Second "Teaser" (High Energy)
*   **Goal:** Capture immediate attention on social media platforms like LinkedIn or Twitter.
*   **Storyboard:**
    *   **0-5s:** Cinematic opening showing the "Metal Wheels Logo" fading in on a dark glass background.
    *   **5-15s:** Rapid cuts of the BMW M8, Rolls Royce, Porsche, and Lamborghini video cards looping smoothly.
    *   **15-25s:** Showcase the interactive hover actions, brand nav buttons, and the interactive Google Map.
    *   **25-30s:** Call-to-action: "Explore the Code on GitHub | Link in Comments".

### B) The 2-Minute "Walkthrough & Code Review" (Professional)
*   **Goal:** Pitch your development skills directly to technical recruiters, portfolio reviewers, and senior developers.
*   **Storyboard:**
    *   **0-30s:** Introduce yourself, explain that this is your **first project from 2023 (first year B.Tech) built entirely by hand, without any AI assistance**. Highlight this proudly!
    *   **30-70s:** Demo the web interface live. Show the smooth transitions, clean font combinations, and responsive card lists.
    *   **70-100s:** Split-screen showing your clean source code side-by-side with the rendered elements. Show off your semantic tags (like `<header>`, `<nav>`, `<section>`, `<footer>`) and CSS hover states.
    *   **100-120s:** Talk about what you learned (layouts, HTML form lifecycle, iframe embeds) and your growth since.

## 4. Best Upload & Hosting Locations

To reach the right audience, host your media where they will load instantly and display with maximum premium quality.

### A) Where to Upload Images

1.  **GitHub Assets (Recommended for repo):**
    *   *How:* Create a folder `docs/images/` inside the repo, commit the screenshots, and use relative markdown paths.
    *   *Pros:* Ensures screenshots stay with the repository permanently and render perfectly in the `README.md`.
2.  **Imgur or Cloudinary (Recommended for external sharing):**
    *   *Pros:* Extremely fast CDN. Provides permanent direct image links that load instantly on portfolio websites or resume pages.
3.  **Your Portfolio Site:**
    *   Embed them in a "Metal Wheels" portfolio card using modern CSS grid layouts with lazy-loading.

### B) Where to Upload Videos

1.  **GitHub Releases (Perfect for the Repo):**
    *   You can attach your 2-minute walkthrough MP4 to your GitHub Release tags. Users can download and watch it locally.
2.  **YouTube (Perfect for embedding):**
    *   *Why:* Hosting on YouTube lets you embed the video directly in your portfolio and GitHub README using an interactive player.
    *   *Tip:* Set it as "Unlisted" if you only want people who visit your portfolio to see it, or "Public" to rank in search results.
3.  **LinkedIn (Perfect for Networking):**
    *   *Why:* Native video on LinkedIn has massive organic reach. Write a post sharing your B.Tech journey, tag your college, and talk about coding from scratch in 2023. This is an incredibly strong post for recruiter visibility!
4.  **Google Drive / OneDrive (Backup):**
    *   Keep a shared link of the raw videos in your resume for easy download.
